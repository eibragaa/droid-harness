# Droid Harness Mobile Status - 2026-05-12

## Implementado

- App Flutter criado em `mobile/`
- Interface mobile para:
  - chat com modelo local via `http://127.0.0.1:8080/v1/chat/completions`
  - health check do `llama-server` via `/v1/models`
  - terminal controlado pelo app
  - comandos rapidos para `llama-server`, modo pouca RAM, health check e proot Ubuntu
- Bridge Termux em `scripts/termux-bridge.py`
- Script de boot do bridge em `scripts/start-termux-bridge.sh`
- Bridge testado localmente com:
  - `GET /health`
  - `POST /terminal/session`
  - `POST /terminal/input`
  - `GET /terminal/events`
- APKs gerados por ABI para reduzir tamanho.

## Como Testar No Celular

No Termux, dentro do repo:

```bash
scripts/start-termux-bridge.sh
```

Depois abrir o app. O cabecalho deve mostrar:

- `Termux online`, quando o bridge estiver rodando
- `LLM online`, quando o `llama-server` estiver ativo em `127.0.0.1:8080`
