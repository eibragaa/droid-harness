# Droid Harness Mobile Status — 2026-05-12

## Implementado (MVP + v1.3)

### App Flutter
- Flutter app em `mobile/`
- **Material 3 Dark** com seed teal `#80cbc4` (paleta Google AI Edge Gallery)
- Design system unificado em `AppColors`
- Dois painéis principais: **Chat** e **Modelo** (NavigationBar)
- Chat com bolhas de 24px radius, avatares (Edge Gallery style)
- Model Manager: card com nome, perfil, contexto, ngl, status, botão de ação
- Bottom sheet modal para terminal (arrastável 40%-90%)
- Barra de status com dots + glow (LLM online/offline + Bridge)
- Model chip no AppBar

### Bridge Termux
- `scripts/termux-bridge.py` — servidor HTTP com endpoints:
  - `GET /health`, `GET /hardware`
  - `POST /terminal/session`, `POST /terminal/input`
  - `GET /terminal/events?after=N`, `POST /terminal/stop`
  - `POST /llm/start`, `POST /models/download`
- `scripts/start-termux-bridge.sh` — script de inicialização
- `scripts/model-profile.sh` — detecção de hardware com perfis weak/balanced/strong
- `scripts/install-termux-boot.sh` — helper para Termux:Boot

### Conexão automática
- Retry automático com até 25 tentativas (intervalo de 3s)
- Cria sessão PTY automaticamente ao conectar (`POST /terminal/session`)
- Detecta hardware e sugere modelo recomendado
- Botão "Baixar + Iniciar" com timeout de 10 min para download

## Como testar

No Termux:
```bash
cd ~/droid-harness
bash scripts/start-termux-bridge.sh
```

No app:
1. Abrir — conecta automaticamente ao bridge (até ~75s)
2. Hardware detectado → botão "Baixar + Iniciar" aparece
3. Tocar para baixar modelo e iniciar llama-server
4. Após online, digitar prompts no chat
